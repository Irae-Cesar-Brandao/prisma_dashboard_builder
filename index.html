📘 PRISMA Dashboard Builder Guide

This document explains each menu and feature of the PRISMA Dashboard Builder system.

1. Project

This section defines the main information of the systematic review.

  • Project Title: Name of the systematic review.
  • Authors: List of researchers involved in the study.
  • Research Period: Defines the search time window (start and end dates).

2. Databases

This area is used to register where the literature searches were performed.

  • Allows adding multiple databases (e.g., PubMed, Scopus, Web of Science).
  • + Add Database: Inserts new search sources.

3. Project Management

Controls project saving and data handling features:

  • Save Project: Stores data locally in the browser.
  • Load Project: Retrieves a previously saved project.
  • Delete Project: Removes stored project data.
  • Backup JSON: Exports a full backup of the project.
  • Import JSON: Loads a previously exported backup file.

4. Initial Screening Removal

Stage for cleaning and filtering data before screening.

  • Duplicates: Number of duplicate records removed.
  • Out of Time Range: Studies outside the selected period.
  • Not Related: Studies not relevant to the topic.

5. Screening

Stage where studies are evaluated based on title and abstract.

  • Screening Excluded: Number of articles removed during initial screening.

6. Eligibility

Stage where full-text articles are assessed for eligibility.

  • Eligibility Excluded: Number of studies excluded after full-text review.

7. Output Generation

  • Generate Dashboard: Creates the PRISMA flow diagram automatically.
  • Generate Report: Produces a structured systematic review report.
  • Export Report: Exports the report as a TXT file.
  • Export Dashboard PDF: Generates a PDF version of the PRISMA diagram.

8. PRISMA Result

Displays the visual PRISMA flow diagram representing all stages of the systematic review process.

9. PRISMA Report

Shows the automatically generated textual report based on the input data.
